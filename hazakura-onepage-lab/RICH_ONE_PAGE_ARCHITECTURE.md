# 葉桜ラボ 1ページ拡張アーキテクチャ

このサイトは「ページ数を増やす」のではなく、1ページの中で体験密度を上げ続ける実験場として育てる。

## 基本方針

- `index.html`: 物語の順番と大きな舞台装置だけを置く。
- `content.js`: 書庫、制作物、構想、数値、体験レイヤーなど、増減する情報を集約する。
- `script.js`: コンテンツ描画、スクロール、粒子、ゾーン遷移などの振る舞いを担当する。
- `style.css`: セクション、カード、テーマ、レスポンシブの見た目を担当する。

## 1ページ内のレイヤー

1. Narrative: RFPにある思想を、ヒーローから哲学へ自然に読ませる。
2. Architecture: 1ページを増築できる構造そのものを見せる。
3. Library: 出版物と問いを見せる。
4. Works: 道具、アプリ、映像構想など制作物を蓄積する。
5. Vision: AI寺子屋、自分史、Research Logなど将来接続する入口を置く。
6. Atmosphere: 昼、夕、夜、月、オーロラの演出でページ全体の空気を変える。

## 次に伸ばしやすい場所

- 制作物を増やす: `content.js` の `projects` に項目を追加する。
- 構想カードを増やす: `content.js` の `visions` に項目を追加する。
- 演出の段階を増やす: `script.js` の `updateScrollZones` と CSS の `body.zone-*` を追加する。
- CMS化する: `docs/CONTENT_DATA_MAP.md` の依存表を見ながら `content.js` と同じ形の JSON を生成し、読み込み元だけを差し替える。

## ルーチン運用

日々の小さな改善は、ルートの `docs/` で管理する。

- `../docs/IMPROVEMENT_LOOP.md`: 1回の改善作業の型。
- `../docs/RICHNESS_BACKLOG.md`: 小さく切った改善候補。
- `../docs/RICHNESS_LOG.md`: 実施したリッチ化の記録。

## 守ること

- HTMLへカードを直書きしない。増えるコンテンツは `content.js` に寄せる。
- セクションの並びを変える時は、ナビゲーションと `data-zone` も同時に見る。
- 見た目の演出はコンテンツの意味を追い越さない。濃くする時ほど、テキストの可読性を最優先にする。
