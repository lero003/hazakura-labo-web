# 🪄 葉桜ラボ Webサイト制作 RFP (Master Version)

## 〜AIの知性と、自然の情緒が溶け合う「循環する魔法の拠点」〜

## 1. プロジェクト概要 (Project Overview)
* **プロジェクト名**: 葉桜ラボ Webサイト構築プロジェクト
* **背景**: AIを用いた探求（Lab）と、東洋的な情緒（葉桜）を融合させ、「変化を楽しむ」という思想を世界へ届ける。さらに、地域社会の課題を「道楽」で解決する「AI万屋」のプラットフォームとしての役割も担う。
* **目的**: 
    1. **探求の可視化**: 葉桜ラボの研究・実験（Research Log）と魔法の道具（Magic Tools）の公開。
    2. **知恵の継承**: 出版物（Library）や、地域の人々の「自分史」を次世代へ繋ぐ。
    3. **循環の創出**: 「AI寺子屋」や「AI万屋」を通じて、地域の知恵と価値が回るエコシステムの入り口となる。
    4. **ブランド確立**: 「AI×情緒」「理×情」という独自のブランドイメージを世界へ発信する。

## 2. コア・フィロソフィー (Core Philosophy)
* **思想**: 
    * **変化の肯定（ゆらぎ）**: 佐久間象山の歌に倣い、満開だけでなく「移ろい（葉桜）」の中に価値を見出す。
    * **理と情の融合（習合）**: 西洋的な論理（AI/Lab）と東洋的な情緒（自然/大和言葉）を対立させず、溶け合わせる。
    * **知恵の循環（事上磨錬）**: 使うほどに賢くなり、教え合うことで豊かになる「生きているシステム」。
* **デザインコンセプト**: 「自然の生命力」×「デジタルな知性」。透明感、光、そして心地よい「間（ま）」を感じさせる動き。

## 3. コンテンツ詳細設計 (Detailed Content Plan)

### 🧪 Research Log (研究記録：探求のプロセス)
* **形式**: 「問い $\rightarrow$ 実験 $\rightarrow$ 発見」のサイクルで記述。
* **テーマ例**: AIの思考プロセス、未知の論文からの概念抽出、日常の哲学的な驚き（thaumazein）。
* **トーン**: 知的な冒険の記録。

### ✨ Magic Tools / Experiments (実験室：魔法の道具)
* **紹介するもの**: `arxiv_scout`, `hermes-agent` など。
* **見せ方**: 「何ができるか」だけでなく、「なぜ作ったのか（動機）」と「どんな驚きがあったか」をセットにする。

### 📚 Library (書庫：知恵の結晶)
* **メイン作品**: 『チカちゃんの哲学冒険譚』シリーズ。
* **構成要素**: 表紙、あらすじ、扱っている「問い」、Amazonリンク。

### 🌸 AI万屋 & 寺子屋 (Community: 道楽と循環)
* **AI寺子屋**: 子どものためのコーチング学習、シニアのための「逆転の知恵（教える側）」体験。
* **自分史編纂**: 人々の思い出を物語として紡ぎ、デジタルアーカイブ化するプロセス。
* **レトロ・ガジェット再生**: 古いモノにAIの魂を宿すワークショップの記録。

## 4. 機能要件 (Functional Requirements)
* **レスポンシブ対応**: 全デバイスで美しく表示されること。
* **インタラクティブな演出**: マウス操作やスクロールに反応する「魔法のような動き（Framer Motion等）」。
* **循環型CMS**: けい君が簡単にログを更新でき、かつそのデータが「地域LLM」の学習へと繋がる構造。
* **コミュニティ・インターフェース**: ユーザーが「問い」を投げかけたり、自分の「物語」を預けたりできる窓口機能（将来的な拡張）。

## 5. 技術スタック案 (Tech Stack)
* **Frontend**: Next.js / React / Tailwind CSS (高速でモダンな体験)
* **Animation**: Framer Motion (ゆらぎのある動きの実現)
* **Deployment**: Vercel または Cloudflare Pages
* **Backend/Data**: Local-SLM連携、ベクトルデータベース（知恵の蓄積用）

## 6. 開発ロードマップ (Roadmap)
* **Phase 1: プロトタイプ構築** (主要ページのデザインと基本機能の実装)
* **Phase 2: コンテンツ投入** (本、ログ、ツールのデータ入力 + AI万屋のコンセプト導入)
* **Phase 3: 魔法の演出追加** (アニメーションやインタラクションの強化 + 循環の仕組み設計)
* **Phase 4: 本番公開！**
